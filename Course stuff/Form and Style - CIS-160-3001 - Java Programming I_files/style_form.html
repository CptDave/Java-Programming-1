Program Style Standards for CIS 160

The following coding style must be followed for this class. Please note how points will be deducted for the various infractions.

Breaking conventions will receive a warning on the first few assignments. After the first several assignments points will be deducted for not following the conventions. Point deductions will increase as the semester progresses.

Each program must have comments at the top of the page that include:

  • The name of the file
  • The course
  • Your name
  • The assignment number
  • The due date
  • A brief description of the program.

Example (using single line comments):

//  sampleProgram.cpp
//  CIS 160 3001
//  Dave Klick
//  Programming Project #
//  2016-09-15
//  This program adds numbers entered by the user.

Example (using a block comment):

/*
    sampleProgram.cpp
    CIS 160 3001
    Dave Klick
    Programming Project #
    2016-09-15
    
    This program adds numbers entered by the user.
*/

Identifiers

  • All identifiers should be descriptive. If you abbreviate, it should be a recognizable abbreviation.
    For example: numOfItems   or numItems instead of: ni
  • Variable and method names should start with a small letter.
  • Identifiers that consist of several words should use camel casing where the first letter of each succeeding word in the identifier is capitalized.
    For example: grossSales
    You may also use underscores to seperate words in an identifier.
    For example: gross_sales
  • Utility variables such as loop counters or flags that do not have meaning within the program scope may be single letters or abbreviations.

Variables

  • All variables should be declared in an orderly manner at the start of a method. Utility variables (counters, flags, etc.) may be declared in the program where they are used.
  • Keep the types grouped together in one declaration. If the list becomes long, start a new line to allow all variables to be read without scrolling.
  • Java requires you to initialize local variables before using them. If you do not, you will get a compiler error stating that the variable might not have been initialized.
  • Do not put multiple statements on one line.
    For example use:
    int numOfStudents, numOfScores;
double test1, test2, test3, testAvg;
String strTest;
    instead of: 
    int numOfStudents, numOfScores; double test1, test2, test3, testAvg; String strTest;
  • You may not use ANY global variables in your programs. All variables must be local variables. Any assignments submitted that use global variables will be given a score of 0.

Constants

  • Constants should be declared in ALL capital letters. Use underscores to separate words in a constant. For example:
    final int SIZE = 20;
final double TAX_RATE = .0675;

Methods

  • All method names should start with a small letter. Each successive word in the identifier should be capitalized. The name should help identify that method's purpose.
    For example: getSalesInfo
  • Method names should include verbs that help describe what they do. For instance use getInput instead of input.
  • Each method should be preceded by a short comment that describes the action performed by the method.

Indentation and curly braces

  • Opening curly braces placed on the same line as the if, switch, while, for or method header. A space should precede the curly brace in this case. The closing curly brace must be aligned with the beginning of the text on the line the opening curly brace is on.
    For example:
    if (x < 20) {
    statements here.......
}
    OR 
    public static void main(String[] args) {
   ....
   return 0;
}
  • Opening curly braces may be placed below the first letter of the if, switch, while, for or method header. The closing curly brace must be aligned with the opening curly brace.
    For example:
    if (x < 20)
{
    statements here.......
}
    OR 
    public static void main(String[] args)
{
   ....
   return 0;
}
  • All statements inside a set of curly braces should be indented 4 spaces.
  • Spaces must be used for indentation - NOT tabs
  • Decision and loop structures with one statement and no curly braces must contain the the statement on the same line as the decision or loop structure.
    For example:
    if (x > 20) x = 0;
while (x < 20) x++;

White space

  • A blank space between operators and operands is optional, but usually preferred (except for increment and decrement).
  • An extra line between structures (decision or loop blocks) helps delineate the structure when reading code.
  • Do not double space code.
  • A blank line between methodss is a good idea to separate the blocks of code.
  • Do not add multiple blank lines between statements.

Comments

  • Documentation comments at the start of the code are mandatory for this course.
  • A comment before each method stating its purpose is also mandatory.
  • You should add other comments in the program where you think it is appropriate to document what the code is doing.
  • Do not add simplistic comments that just serve to clutter code (such as: a + b; // add a and b)
